## 使用建议

1. **记录创作过程**
   - 在 `questions/` 记录为什么要写这个主题
   - 在 `workflows/writing.md` 记录使用的工作流
   - 标注 LLM 辅助的程度

2. **保持版本控制**
   - 大的修改提交 Git
   - 可以使用分支进行实验性写作
   - 记录迭代过程

3. **质量把控**
   - LLM 生成的内容需要人工审核
   - 特别注意事实准确性
   - 保持个人风格和观点

4. **知识沉淀**
   - 好的内容可以分享
   - 提取可复用的 Prompt
   - 总结写作经验到 `workflows/`
