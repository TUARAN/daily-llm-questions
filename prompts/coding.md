# Coding Prompts

## 代码审查

```
请审查以下代码：

[粘贴代码]

关注点：
1. 潜在的 bug 或边界情况
2. 性能优化建议
3. 代码可读性和维护性
4. 是否符合最佳实践

请具体指出问题所在，并提供改进建议。
```

## 功能实现

```
需求：[详细描述功能需求]

技术栈：[使用的语言、框架]

约束条件：
- [性能要求]
- [兼容性要求]
- [其他限制]

请提供：
1. 实现思路和架构设计
2. 关键代码片段
3. 需要注意的边界情况
4. 测试建议
```

## 调试辅助

```
遇到了一个问题：

错误信息：
[粘贴错误信息]

相关代码：
[粘贴相关代码]

已尝试的方法：
- [方法1]
- [方法2]

请帮我：
1. 分析可能的原因
2. 提供调试思路
3. 建议解决方案
```

## 代码重构

```
需要重构的代码：

[粘贴代码]

重构目标：
- [ ] 提高可读性
- [ ] 减少重复
- [ ] 优化性能
- [ ] 更好的扩展性

请提供重构方案，并解释改进的地方。
```

## API 设计

```
需要设计一个 API 用于：[功能描述]

请帮我设计：
1. 合理的端点（endpoints）
2. 请求/响应数据结构
3. 错误处理方式
4. 认证和权限方案

遵循 RESTful 原则，考虑可扩展性。
```

## 使用建议

- 提供足够的上下文和需求
- 代码审查时说明语言和框架版本
- 不要盲目采用生成的代码，理解后再使用
- 记录好用的 Prompt 和改进点
