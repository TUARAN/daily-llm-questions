# 研究工作流

> 如何用 LLM 辅助深度研究和学习

## 研究流程

### 1. 主题确定和范围界定

**目标：**
- 明确研究问题
- 界定研究范围
- 识别关键概念

**LLM 辅助：**

```
我想研究：[主题]

请帮我：
1. 拆解这个主题的核心问题
2. 建议研究的范围和边界
3. 列出关键概念和术语
4. 提供研究框架建议
```

---

### 2. 文献调研

**目标：**
- 了解现有研究
- 识别知识空白
- 建立知识地图

**工作方式：**
1. 搜索和收集资料
2. 快速阅读和筛选
3. 深度阅读重点文献
4. 整理和总结

**LLM 辅助：**

```
我读了以下材料：
[粘贴摘要或关键点]

请帮我：
1. 提取核心观点
2. 识别共识和争议
3. 指出可能遗漏的角度
4. 建议下一步阅读方向
```

---

### 3. 深度理解和整合

**目标：**
- 建立完整的知识体系
- 发现内在联系
- 形成自己的见解

**思考问题：**
- 这些观点之间有什么关系？
- 为什么会有不同的看法？
- 我的理解是否准确？
- 还有什么未解决的问题？

**LLM 辅助：**

```
我对 [主题] 的理解框架是：

[描述你的理解]

请：
1. 评估这个框架的完整性
2. 指出可能的盲点
3. 建议补充的维度
4. 提供不同的理解框架对比
```

---

### 4. 应用和实践

**目标：**
- 将知识转化为能力
- 通过实践加深理解
- 发现新的问题

**方式：**
- 写文章讲解
- 做实验验证
- 应用到实际项目
- 教授他人

**LLM 辅助：**

```
我想通过 [方式] 来实践对 [主题] 的理解。

计划：
[描述计划]

请：
1. 评估这个计划的合理性
2. 建议需要注意的点
3. 提供实践的具体步骤
4. 推荐评估效果的方法
```

## 使用技巧

### 1. 始终保持批判性思维

- LLM 提供的信息需要验证
- 不要被单一观点局限
- 交叉验证多个来源

### 2. 用 LLM 加速，但不替代思考

- 用 LLM 快速获取概览
- 深度理解靠自己思考
- 用 LLM 验证和补充

### 3. 记录研究过程

- 记录阅读的材料
- 记录思考的演进
- 记录失败的假设

### 4. 建立知识连接

- 新知识如何关联旧知识？
- 如何应用到不同领域？
- 什么是通用原理？
