# 写作工作流

> 如何用 LLM 辅助写作，同时保持个人风格

## 核心原则

1. **LLM 是工具，不是替身**
   - 思想来自于你
   - 结构由你把控
   - 风格是你的

2. **写作是思考的过程**
   - 不要跳过思考直接让 LLM 生成
   - 用 LLM 辅助思考，而非替代思考

3. **迭代优化**
   - 先有初稿，再优化
   - 多次小步迭代，而非一次完美

---

## 写作流程

### 阶段1：构思（30%时间）

**目标：**
- 明确写作目的
- 确定目标读者
- 梳理核心观点
- 设计文章结构

**自己完成：**
1. 确定主题和观点
2. 列出要点
3. 思考逻辑

**LLM 辅助：**

```
我想写：[主题]

我的核心观点：
1. [观点1]
2. [观点2]

目标读者：[描述]

请帮我：
1. 评估这些观点是否连贯
2. 建议文章结构
3. 指出可能需要补充的内容
4. 推荐合适的例子或数据

不要直接写内容，只提供结构建议。
```

**输出：**
- 文章大纲
- 每部分要点
- 需要的素材清单

---

### 阶段2：初稿（40%时间）

**目标：**
- 快速完成初稿
- 不追求完美
- 保持流畅

**工作方式：**
1. 按大纲展开每个部分
2. 先写核心内容，细节后补
3. 不要停下来纠结用词
4. 保持写作节奏

**LLM 辅助（谨慎使用）：**

```
我在写 [章节]，要表达的是：[核心意思]

请提供2-3种不同的表达方式，帮我打开思路。
不要直接给我成品，我需要在此基础上修改。
```

**注意：**
- 不要完全依赖 LLM 生成内容
- LLM 生成的内容要用自己的语言重写
- 保持个人风格和表达习惯

---

### 阶段3：修改和优化（30%时间）

**目标：**
- 优化结构和逻辑
- 提升表达质量
- 增强可读性

**自己完成：**
1. 通读全文，检查逻辑
2. 调整结构，确保流畅
3. 删除冗余，突出重点

**LLM 辅助：**

#### 3.1 结构检查

```
文章内容：
[粘贴全文或大纲]

请评估：
1. 结构是否合理
2. 逻辑是否连贯
3. 重点是否突出
4. 是否有冗余部分

只提供分析和建议，不要重写。
```

#### 3.2 语言优化

```
请帮我优化这段文字：

[粘贴段落]

要求：
- 保持原意
- 更简洁/更生动/更准确
- 保持原有风格

提供修改建议并说明理由。
```

#### 3.3 可读性检查

```
请评估这篇文章的可读性：

[粘贴部分内容]

从以下方面评估：
1. 句子长度和复杂度
2. 段落组织
3. 过渡是否自然
4. 是否容易理解

提供具体的改进建议。
```

## 不同类型文章的策略

### 技术文章

**重点：**
- 准确性 > 文采
- 结构清晰
- 代码示例正确

**LLM 使用：**
- 检查技术准确性
- 生成代码示例
- 优化解释方式

### 观点文章

**重点：**
- 观点鲜明
- 逻辑严密
- 论证充分

**LLM 使用：**
- 寻找论据
- 反驳预设
- 检查逻辑漏洞

### 教程文章

**重点：**
- 步骤清晰
- 易于跟随
- 考虑新手视角

**LLM 使用：**
- 检查步骤完整性
- 识别可能的困惑点
- 优化说明文字

## 写作技巧

### 1. 开头

**目的：**
- 吸引读者
- 建立预期
- 引出主题

**Prompt：**
```
文章主题：[主题]
核心内容：[简述]

请提供3种不同风格的开头方式：
1. 故事型
2. 问题型
3. 直接型

每种提供思路，不要直接写成品。
```

### 2. 过渡

**目的：**
- 连接段落
- 保持流畅
- 引导思路

**技巧：**
- 承上启下
- 使用连接词
- 提出问题

### 3. 结尾

**目的：**
- 总结要点
- 升华主题
- 引发行动/思考

**Prompt：**
```
文章表达了：[核心观点]

请建议几种结尾方式，要求：
1. 呼应开头
2. 总结升华
3. 给读者启发

只要思路，不要写成品。
```

## 常见陷阱

### 1. 过度依赖 LLM

**表现：**
- 直接用 LLM 生成内容
- 不经思考就采用建议
- 失去个人风格

**解决：**
- 先有自己的想法
- LLM 建议要筛选和改写
- 保持写作的主动权

### 2. 追求完美

**表现：**
- 反复修改一个段落
- 不敢发布
- 迟迟不能完成

**解决：**
- 先完成，再完美
- 设置时间限制
- 接受"足够好"

### 3. 忽略读者

**表现：**
- 自说自话
- 术语过多
- 假设读者知道一切

**解决：**
- 明确目标读者
- 多用例子和类比
- 让朋友试读
