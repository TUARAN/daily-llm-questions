# Quick Start Guide

## 今天就开始使用

### 1️⃣ 记录第一个问题（2分钟）

打开 [questions/2026-01.md](questions/2026-01.md)，添加你今天遇到的第一个问题：

```markdown
## 2026-01-20

### 问题：[你的问题]

**背景：**
[为什么问这个问题]

**回答：**
[AI 的回答或你的发现]

**总结：**
[学到了什么]
```

### 2️⃣ 选择一个工作流（5分钟）

根据你今天的主要工作，选择一个工作流开始实践：

- 📝 **写作？** → [workflows/writing.md](workflows/writing.md)
- 💻 **编码？** → [workflows/daily.md](workflows/daily.md) 
- 🔍 **研究学习？** → [workflows/research.md](workflows/research.md)

### 3️⃣ 使用 Prompt 模板（随时）

当你需要问 AI 时，从这里找合适的模板：

- [prompts/thinking.md](prompts/thinking.md) - 深度思考
- [prompts/writing.md](prompts/writing.md) - 写作辅助
- [prompts/coding.md](prompts/coding.md) - 编码相关

## 常见场景

### 场景1：快速提问

1. 打开当月的 questions 文件
2. 添加问题和回答
3. 继续工作

### 场景2：系统性思考

1. 使用 [prompts/thinking.md](prompts/thinking.md) 中的模板
2. 在编辑器里与 AI 对话
3. 将有价值的对话记录到 questions/

### 场景3：写作项目

1. 按照 [workflows/writing.md](workflows/writing.md) 的流程
2. 使用 [prompts/writing.md](prompts/writing.md) 的模板
3. 在项目文件夹里迭代

### 场景4：实验新想法

1. 在 [experiments/](experiments/) 创建新文件或使用现有文件
2. 记录实验过程和结果
3. 失败也要记录到 [experiments/failures.md](experiments/failures.md)

## 技巧

### ✅ 推荐做法

- **真实记录**：不追求完美，追求真实
- **及时记录**：问题和答案产生时就记录
- **持续迭代**：发现好用的 Prompt 就加入到模板
- **保持简单**：不要让记录成为负担

### ❌ 避免陷阱

- 不要只收藏不使用
- 不要追求完美的格式
- 不要把这当作"作品集"
- 不要让工具使用复杂化
